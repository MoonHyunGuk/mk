[본사]
1. 자료업로드(GR15)
	1) 파일이 있으면 처리안됨(하루 한건만 입력가능)
	2) GR15 파일 저장
	2) Header Record 처리
	3) Data Record 처리
		A) 고객조회번호 맨 앞 2자리가 99이면 신 고객조회번호, 99가 아니면 구 고객조회번호로 구분
		B) 신 고객조회번호 처리 ( 구분(2) + 매체코드(2) + 독자번호(9) + 시작년월(4) + 개월분(2) + check(1) )
			a) 매체코드,독자번호,시작년월,종료년월,지국코드를 구한다.
		C) 구 고객조회번호 처리 ( 지국코드(6) + 구역(2) + 지국독자번호(5) + 시작월(2) + 종료년월(4) + check(1) )
			a) 매체코드(default:100),지국코드,구역코드,지국독자번호,시작년월,종료년월을 구한다.
		D) 수납구분이 Reject 이거나 장표처리구분이 Reject 이면 오류처리
		E) EDI 테이블에 row data 저장
		F) 위에 파싱한 정보들이 잘못되거나 에러가 아니면 입금처리
			a) 해당 정보에 매칭되는 수금목록 추출
			b) 지로로 받은 금액이 미수금액과 같으면 미수 전체 업뎃
			c) 지로로 받은 금액이 미수금액보다 크면 미수정보 일괄 수금처리 후 남은 금액 선입금 처리(딱 떨어지지 않거나 구독정보없거나 중지독자면 과입금처리)
			d) 지로로 받은 금액이 미수금액보다 작으면 과거건부터 순차 입금처리
			e) 미수금액이 없고 구독정보가 있으면서 중지독자가 아니면 종료년월까지 선입금처리 후 나머지 과입금처리
	4) Trailer Record 처리
	5) FILE_T 테이블에 GR15 파일이름 저장

2. 자료조회
	- 일자별 EDI 입금내역 조회

	- 일자별 EDI 입금오류내역 조회

	- 일자별 EDI 과입금내역 조회
